# Contributing to Awesome AI Knowledge

Thank you for considering contributing to Awesome AI Knowledge! 

## Guidelines

### Adding a New Item

1. **Ensure the item is relevant** - It must be a knowledge base tool, framework, or engine for AI agents.
2. **Check for duplicates** - Make sure the item isn't already listed.
3. **Follow the format** - Use the following format:
   ```
   - [Project Name](https://github.com/user/project) - Brief description.
   ```
4. **Add in the correct category** - Place the item in the appropriate section alphabetically.
5. **Submit a pull request** - With a clear title and description.

### Item Quality Standards

- **Active development** - The project should have recent commits (within the last 6 months).
- **Documentation** - Good README with installation and usage instructions.
- **Open source** - Must be open source with a recognized license.
- **Working state** - The project should be functional and usable.

### Categories

- **Knowledge Base Engines** - RAG frameworks, knowledge base platforms, AI-powered search engines.
- **Knowledge Graph Tools** - Tools for building, managing, and querying knowledge graphs.
- **Vector Databases** - Databases optimized for storing and querying vector embeddings.
- **MCP Knowledge Services** - Model Context Protocol servers and tools for knowledge management.
- **LLM Wiki Tools** - Tools for building and managing wiki-style knowledge bases for LLMs.
- **Document Processing & Indexing** - Tools for parsing, chunking, and indexing documents.

### Description Guidelines

- Keep descriptions concise (under 100 characters).
- Start with a capital letter.
- End with a period.
- Use present tense.
- Don't start with "A" or "An".

### Removal

If you believe an item should be removed, please open an issue with:
- The item name and link
- Reason for removal (e.g., unmaintained, no longer relevant, replaced by better alternative)

## Code of Conduct

Be respectful and constructive in all interactions. We're here to build a helpful resource together.
